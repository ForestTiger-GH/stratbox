# Strategy Box (`stratbox`)

`stratbox` — Python-библиотека для прикладной работы с данными в Jupyter Notebook, Google Colab, локальных скриптах и расширенной корпоративной среде. Библиотека объединяет инфраструктурный слой для чтения, записи и хранения файлов с доменными инструментами для макроэкономических и банковских задач.

Strategy Box используется как набор инструментов для задач: обработки банковской отчетности, форм Банка России, Excel/CSV/DBF-файлов, подготовки витрин данных и прикладных расчетов по макроэкономике и банковскому сектору.

Ключевые направления: формы ЦБ РФ 101, 102, 135, 805; банковская отчетность; макроэкономические данные; Jupyter Notebook; Google Colab; корпоративные файловые хранилища.

Главная идея проекта — дать пользователю единый и предсказуемый интерфейс, который одинаково работает в двух режимах:

- **стандартная среда**: локальный диск, Colab, обычный Jupyter;
- **расширенная корпоративная среда**: сетевой диск, внутренние шлюзы, секреты и дополнительные провайдеры.

Пользовательский код при этом должен оставаться максимально одинаковым: доменные модули не должны напрямую знать, где физически лежит файл — локально, в `/content`, на сетевом диске или в другом подключенном хранилище.

---

## Быстрый старт

Базовый импорт инфраструктурного IO-слоя:

```python
from stratbox.base import ioapi as ia
```

Запись DataFrame в Excel:

```python
ia.excel.write_df("outputs/table.xlsx", df)
```

Чтение Excel:

```python
df = ia.excel.read_df("outputs/table.xlsx")
```

Работа с доменами `macrobanks`:

```python
from stratbox.macrobanks.cbr_forms import run_all_forms_to_xlsx
from stratbox.macrobanks.escrow import run_escrow_to_xlsx
from stratbox.macrobanks.frg import run_frg_stage1, run_frg_cleanup
from stratbox.macrobanks.cbr_archiver import run_cbr_archiver
```

## Установка

### Стандартная среда

```bash
pip install -e .
```

### Расширенная корпоративная среда

В корпоративной среде устанавливается сам `stratbox` и отдельный плагин, который добавляет корпоративные провайдеры:

- файловый транспорт для сетевого диска;
- провайдер секретов;
- корпоративные Excel-пресеты;
- обработку URL через внутренние шлюзы, если это настроено в плагине.

Сам `stratbox` не содержит корпоративных адресов, учетных данных и прямых импортов плагина. Подключение выполняется через entry-points.

---

## Desktop shell внутри репозитория

Внутри репозитория есть пакет `src/app` — desktop-приложение Strategy Box, публикуемое как AppDock app target. Оно не заменяет библиотеку `stratbox`, а использует её как доменное ядро.

Главный маршрут такой:

- AppDock подготавливает node-среду;
- AppDock создаёт session surfaces и handoff;
- `python -m app.entrypoints.appdock` читает этот контракт;
- GUI запускает задачи поверх business-root, который уже выбран AppDock.

Это разделяет роли:

- AppDock отвечает за install/runtime environment;
- `app` отвечает за session-aware GUI и запуск задач;
- пакет `stratbox` остаётся библиотечным и доменным слоем.

## Архитектурные слои

### `stratbox.base`

Нейтральная инфраструктура библиотеки:

- `filestore` — единый транспорт файлов и каталогов;
- `ioapi` — чтение и запись форматов поверх `FileStore`;
- `net` — единый сетевой слой и обработка URL;
- `runtime` — выбор активных провайдеров: плагин или локальный режим;
- `secrets` — базовый интерфейс получения секретов;
- `styles.excel` — система Excel-пресетов.

### `stratbox.common`

Общие утилиты, которые не привязаны к конкретному домену. Сейчас основной блок — генерация сеток периодов: дни, недели, месяцы, кварталы и годы.

### `stratbox.macrobanks`

Доменный слой для банковских и макроэкономических задач:

- `cbr_forms` — обработка отчетных форм Банка России 101, 102, 123, 135 и 805;
- `escrow` — обработка данных по счетам эскроу;
- `frg` — файловый контур для работы с наборами FRG-файлов;
- `cbr_archiver` — скачивание и сохранение исходных статистических файлов Банка России.

### `stratbox.registries`

Встроенные справочники и реестры: банки, исторические замены, стандартные наименования, ОКВЭД и другие ресурсные таблицы.

## Единый FileStore

Центральная инфраструктурная идея `stratbox` — работа с файлами через `FileStore`.

Доменный код и `ioapi` не должны напрямую зависеть от конкретной файловой системы. Они обращаются к абстрактному файловому транспорту, а физическая реализация выбирается на уровне runtime:

- вне контура — `LocalFileStore`;
- внутри контура — провайдер из плагина, если он установлен и включен.

Поэтому один и тот же вызов может сохранять результат:

- в локальную папку проекта;
- в `/content` в Colab;
- на сетевой диск;
- в другое хранилище, если оно реализовано как `FileStore`.

Пример:

```python
from stratbox.base import ioapi as ia

ia.excel.write_df("outputs/report.xlsx", df)
```

## IO API

Рекомендуемый способ работы с форматами — через `stratbox.base.ioapi`:

```python
from stratbox.base import ioapi as ia
```

Поддерживаемые направления:

- bytes: чтение и запись бинарных файлов;
- Excel: `.xlsx`, `.xlsm`, `.xls`, `.xlsb`;
- CSV;
- DBF;
- XML;
- TXT;
- ZIP/RAR-архивы;
- PDF, DOCX, PPTX и изображения — в пределах реализованных вспомогательных операций.

Важный принцип: форматные модули читают и пишут данные поверх `FileStore`. Это позволяет не размножать отдельный код для локального диска, Colab и корпоративного сетевого диска.

## Excel-оформление

В `stratbox` есть система Excel-пресетов. Она применяется после записи таблицы pandas через `openpyxl` и позволяет централизованно управлять оформлением итоговых файлов.

Базовые блоки форматирования:

1. `VALUES` — обычные ячейки таблицы;
2. `HEADER` — строка или строки заголовков;
3. `FIRST_COLS` — первые столбцы, обычно ключевые поля;
4. `CORNER` — левый верхний угол таблицы, пересечение заголовков и первых столбцов.

Поддерживаются:

- шрифты;
- заливки;
- границы;
- выравнивание;
- перенос текста;
- числовые форматы;
- закрепление областей;
- автофильтр;
- скрытие сетки;
- автоподбор ширины колонок.

Пример:

```python
ia.excel.write_df(
    "outputs/table.xlsx",
    df,
    style_preset="DEFAULT",
)
```

Если установлен плагин, список доступных Excel-пресетов может быть расширен корпоративными стилями.

## Домены `macrobanks`

### CBR forms

Домен `stratbox.macrobanks.cbr_forms` скачивает архивы отчетных форм Банка России, распаковывает DBF, приводит данные к long-формату, затем формирует wide-таблицы и сохраняет отдельный Excel-файл по каждой форме.

Поддерживаемые формы:

- 0409101;
- 0409102;
- 0409123;
- 0409135;
- 0409805.

Пример:

```python
from stratbox.macrobanks.cbr_forms import run_all_forms_to_xlsx

out = run_all_forms_to_xlsx(
    date_from="2024-01-01",
    date_to="2024-03-01",
    forms="101,102,135,805",
    out_dir="outputs/cbr_forms",
)

print(out)
```

### Escrow

Домен `stratbox.macrobanks.escrow` строит единую Excel-витрину по счетам эскроу: получает ссылки на исходные файлы, скачивает Excel, парсит исторически разные форматы публикаций, собирает long-поток и формирует сводные листы по показателям.

```python
from stratbox.macrobanks.escrow import run_escrow_to_xlsx

result = run_escrow_to_xlsx(
    out_path="outputs/Escrow Accounts.xlsx",
    source_cache_dir="cache/escrow",
)
```

### FRG

Домен `stratbox.macrobanks.frg` отвечает за первый файловый этап работы с FRG-файлами: сканирование каталога, распознавание семейств по имени, нормализацию периодов, выбор актуальных файлов, подготовку зачистки каталога и создание ZIP-архива актуального набора.

```python
from stratbox.macrobanks.frg import run_frg_stage1

result = run_frg_stage1("data/frg")
latest_df = result["latest"]
```

### CBR archiver

Домен `stratbox.macrobanks.cbr_archiver` скачивает плоский список статистических файлов Банка России и сохраняет исходники без изменения содержимого: либо отдельными файлами в папку, либо одним ZIP-архивом.

```python
from stratbox.macrobanks.cbr_archiver import run_cbr_archiver

result = run_cbr_archiver(
    out_path="/content",
    output_mode="zip",
)

print(result.output_path)
```

## Практические принципы разработки

1. **Доменный код не должен напрямую импортировать корпоративный плагин.** Взаимодействие с расширенной средой выполняется через `runtime`, `FileStore` и `base.net`.
2. **Чтение и запись файлов должны идти через `FileStore/ioapi`.** Прямые `open()` допустимы только во внутренних временных сценариях, где это осознанно и локально.
3. **Исходные файлы не следует менять без необходимости.** Если домен архивирует или упорядочивает файлы, он должен явно отделять файловые операции от парсинга содержимого.
4. **Расширение справочников и реестров должно быть управляемым.** Для показателей, семейств файлов и источников лучше иметь явные реестры, а не скрытую эвристику в середине пайплайна.
5. **Результаты должны быть воспроизводимыми.** По возможности доменные функции возвращают структурированный результат: путь к выходному файлу, перечень обработанных источников, ошибки и служебные счетчики.

## Desktop shell `app`

Репозиторий содержит пакет `app`, который работает как desktop-оболочка Strategy Box, публикуемая как AppDock app target.

Основной пользовательский маршрут выглядит так:

- AppDock подготавливает node-среду;
- AppDock выбирает target repo, runtime и session surfaces;
- AppDock передает handoff в `python -m app.entrypoints.appdock`;
- `app` строит GUI-контекст от AppDock-managed workspace selector.

`data_root` в этой модели обозначает selector бизнес-среды. Служебные файлы AppDock и install-среды туда не записываются.


## AppDock-managed workspace selector and workspace root

В AppDock-managed режиме приложение различает выбранный shell-ом workspace selector и реальный рабочий каталог. Если selector указывает на корень диска, приложение использует производный каталог `Strategy Box Data` и работает через него как через writable workspace root. Для системного диска используется каталог внутри профиля пользователя, а не корень диска.

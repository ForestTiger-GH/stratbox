# Strategy Box (stratbox)

`stratbox` — библиотека на Python для стандартизации типовых задач анализа данных и прикладного IO (чтение/запись файлов разных форматов) в Jupyter Notebook и в скриптах.

Библиотека задумывается как **единая “коробка инструментов”**:

- единый интерфейс для IO (Excel/CSV/DBF/…)
- единый подход к путям и файловым операциям (каталоги, копирование, удаление)
- поддержка “двух миров”: **вне контура** (локально) и **внутри контура** (через плагин-провайдеры)

---

## Быстрый старт

Рекомендуемый импорт:

```python
from stratbox.base import ioapi as ia
```

Пример записи DataFrame в Excel:

```python
ia.excel.write_df("out.xlsx", df)
```

Пример чтения:

```python
df = ia.excel.read_df("out.xlsx")
```

> Подробная справка по FileStore и методам работы с каталогами: **`docs/files_catalog_methods.md`**.

## Архитектура

### 1) IO API поверх FileStore

Центральная идея — все операции чтения/записи выполняются через единый слой `FileStore`.
Внешне это выглядит просто (`ia.excel.write_df(...)`), но внутри позволяет переключать физическое хранилище:

- **вне контура**: локальный диск (`LocalFileStore`)
- **внутри контура**: сетевое хранилище/SMB/Samba (через `stratbox-plugin`)

### 2) Плагинная модель для “внутреннего контура”

`stratbox` сам по себе не содержит корпоративных драйверов/секретов.  
Если установлен `stratbox-plugin`, то `stratbox` автоматически “подхватывает” провайдеры через entry-points и начинает работать в корпоративном режиме **без изменения пользовательского кода**.

## Excel: стили и форматирование

В `stratbox` реализована система пресетов форматирования Excel, которые применяются **после** записи таблицы pandas’ом (через openpyxl).

Код расположен здесь:

- `src/stratbox/base/styles/excel/`
  - `models.py` — модели пресетов (`StyleSpec`, `BlockStyle`, `FontTheme`)
  - `builtin.py` — встроенные пресеты (внешний режим)
  - `apply.py` — применение стилей к листу
  - `registry.py` / `plugin.py` — подмешивание пресетов из плагина
  - `main.py` — публичные функции (список пресетов, выбор дефолта, apply)

### 4 “кирпичика” таблицы (блоки)

Форматирование построено на 4 блоках (каждый блок — набор опциональных атрибутов):

1. **VALUES** — обычные значения таблицы  
2. **HEADER** — строка(и) заголовков  
3. **FIRST_COLS** — первые N столбцов (часто “ключи”: дата/банк/показатель)  
4. **CORNER** — пересечение HEADER × FIRST_COLS (левый верхний угол)

По умолчанию:
- заголовки = значения (если явно не заданы)
- первые столбцы = значения (если явно не заданы)
- отдельно можно включать/переопределять HEADER/FIRST_COLS/CORNER

В каждом блоке можно задавать:
- заливка, цвет шрифта, жирность
- границы
- выравнивание по горизонтали/вертикали, перенос текста
- **number_format** (даты/проценты/деньги и т.п.)

Шрифты задаются через `FontTheme` (например, Arial 10 / Times New Roman 12), а жирность обычно контролируется через `BlockStyle.bold`.

### Выбор пресета пользователем

По умолчанию используется пресет `"DEFAULT"`:

```python
ia.excel.write_df("out.xlsx", df, style_preset="DEFAULT")
```

Можно выбрать конкретный пресет по имени (из core или из плагина):

```python
ia.excel.write_df("out.xlsx", df, style_preset="COMPANY BLUE")
```

Получить список доступных пресетов:

```python
from stratbox.base.styles.excel.main import list_available_presets, get_default_preset_name
print(list_available_presets())
print(get_default_preset_name())
```

Если передано неверное имя пресета — выбрасывается ошибка с перечнем доступных пресетов (core + plugin, если plugin подключён).

## Excel: автофильтр, freeze panes, сетка, автоширина колонок

Экспорт в Excel реализован в `src/stratbox/base/ioapi/excel_xlsx.py`.

Поддерживается:

- **hide_gridlines** (скрыть/показать линии сетки)
- **freeze panes** (закрепление строк/столбцов)
- **автофильтр** (ставится через openpyxl, без передачи `autofilter_range` в pandas — это важно для совместимости версий)
- **автоподбор ширины столбцов**
  - ширина подбирается автоматически по содержимому и **clamp’ится**: `min_width <= width <= max_width`
  - добавляются паддинги `padding` и `filter_padding` (для столбцов в зоне автофильтра)
  - учитывается кейс “формулы-литералы” вида `=12345` и `="TEXT"` (часто используется, чтобы Excel не портил большие числа)

## Макроэкономика и банки: CBR forms (101/102/123/135)

В `stratbox.macrobanks.cbr_forms` есть готовый пайплайн:

- загрузка реестров/формул
- скачивание и подготовка исходных файлов
- преобразование данных в “long” формат
- формирование “wide” таблиц и экспорт в XLSX

Основной публичный вызов:

```python
from stratbox.macrobanks.cbr_forms import run_all_forms_to_xlsx

out = run_all_forms_to_xlsx(
    date_from="2024-01-01",
    date_to="2024-02-01",
    banks_mode="legacy",
    out_dir=".",
)
print(out)
```

Выводит отдельные файлы `CBR_0409{code}_LEGACY.xlsx` по каждой форме.

## FRG

В `stratbox.macrobanks.frg` реализован первый этап обработки файлов FRG:

- сканирование каталога;
- распознавание семейства файла по имени;
- выбор наиболее свежего файла по каждому семейству;
- диспетчеризация в заглушки будущих парсеров.

Основной публичный вызов:

```python
from stratbox.macrobanks.frg import run_frg_stage1

result = run_frg_stage1("/path/to/frg_files")
catalog_df = result["catalog"]
latest_df = result["latest"]
dispatch_df = result["dispatch"]
```

Подробности: `docs/frg_stage1.md`.

## Установка

### Вне контура (локально)

```bash
pip install -e .
```

### Внутри контура (с плагином)

1) установить `stratbox`  
2) установить `stratbox-plugin` (добавляет корпоративный FileStore, SecretProvider и корпоративные Excel-пресеты)

---

## Структура репозитория (коротко)

- `src/stratbox/base/` — ядро: filestore, ioapi, runtime, utils
- `src/stratbox/base/styles/excel/` — пресеты Excel-стилей (core + plugin overlay)
- `src/stratbox/macrobanks/` — банковские/макро-модули (в т.ч. CBR forms)
- `docs/` — документация пользователя
- `scripts/` — примеры (чтение/запись Excel через ioapi и через pandas)

---

## Совместимость и примечания

- Экспорт Excel использует `openpyxl`. В некоторых окружениях pandas/openpyxl ломаются на `autofilter_range` при записи — поэтому `stratbox` применяет автофильтр вручную через openpyxl.
- Для больших таблиц автоподбор ширины использует ограничение `sample_rows` (ускорение), чтобы не “подвешивать” ноутбук.

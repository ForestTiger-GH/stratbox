# Frank RG — stage 1

Первый этап внутри `stratbox.macrobanks.frank_rg` решает четыре задачи:

1. считывает список файлов из каталога;
2. определяет семейство каждого файла по имени;
3. выбирает наиболее свежий файл по каждому семейству и отправляет его в заглушку будущего парсера;
4. готовит и при необходимости выполняет зачистку каталога без чтения содержимого файлов.

## Публичный API

```python
from stratbox.macrobanks.frank_rg import run_frank_rg_stage1

result = run_frank_rg_stage1("/path/to/frank_rg", recursive=False)
catalog_df = result["catalog"]
latest_df = result["latest"]
dispatch_df = result["dispatch"]
```

## Возвращаемые таблицы

### `catalog`
Полный каталог найденных файлов:
- путь;
- имя файла;
- расширение;
- семейство;
- дата периода;
- номер недели для недельных файлов;
- признаки валидности и причина исключения.

### `latest`
По одному самому свежему файлу на семейство.

### `dispatch`
Журнал маршрутизации в заглушки парсеров.

## Нормализация периода

В stage 1 заложена базовая нормализация периода из имени файла:

- календарные даты приводятся к виду `YYYY-MM-DD`;
- даты с разделителями `-`, `.`, `_`, `/` и смешанными комбинациями распознаются одинаково;
- недельные периоды приводятся к единому виду `YYYY-MM week N`.

## Правило по региональным данным

В коде зафиксирована отсечка по новой региональной схеме:

- региональные семейства считаются валидными только начиная с `2026-04-01`;
- более старые региональные файлы сохраняются в полном каталоге,
  но не попадают в таблицу `latest`.

## Зачистка каталога

Для предварительной работы с файлами добавлен отдельный набор функций:

```python
from stratbox.macrobanks.frank_rg import run_frank_rg_cleanup

result = run_frank_rg_cleanup(
    "/path/to/frank_rg",
    delete_others=False,
    execute=False,
)
catalog_df = result["catalog"]
latest_df = result["latest"]
plan_df = result["plan"]
execution_df = result["execution"]
```

### Принцип работы зачистки

Зачистка работает только по верхнему уровню каталога:

- вложенные папки не сканируются;
- каталог `Архив` и любые другие подпапки не затрагиваются;
- содержимое файлов не открывается и не меняется.

### Что делает режим без удаления

Если `delete_others=False`, то функция:

- определяет наиболее свежий файл по каждому семейству;
- готовит для него новое имя вида `период_family_name.ext`;
- по запросу создает копию этого файла в том же каталоге.

### Что делает режим с удалением

Если `delete_others=True`, то функция:

- переименовывает наиболее свежий файл по каждому семейству;
- удаляет прочие обработанные файлы того же набора;
- удаляет файлы, исключенные только из-за слишком старого периода;
- не удаляет файлы с другими причинами невалидности.

### Выполнение через FileStore

Все операции зачистки идут только через `FileStore` stratbox:

- копирование;
- переименование;
- удаление.

Это нужно для корректной работы как на локальном диске, так и на сетевом хранилище.

### Архивирование итоговых файлов

В `run_frank_rg_cleanup(...)` добавлена отдельная опция архивирования:

- если `archive_latest=True`, то после копирования/переименования создается единый ZIP-архив только с итоговыми файлами `latest`;
- архив сохраняется в том же каталоге верхнего уровня;
- вложенные папки в архив не попадают;
- имя архива формируется как `FrankRG_Actuals_YYYY-MM-DD.zip`;
- если архив уже существует, он полностью перезаписывается.

Архивирование не влияет на распознавание файлов:

- ZIP-файл не участвует в `latest`;
- нераспознанные файлы по-прежнему не затрагиваются;
- чтение содержимого самих Excel-файлов не выполняется, в архив попадают только их байтовые копии.

## Дополнительная логика weekly-файлов

Если в одном и том же месяце присутствует несколько weekly-файлов одного семейства, более ранние недели автоматически считаются устаревшими. Они получают причину `superseded_by_newer_week_in_same_month` и в режиме зачистки могут быть удалены как неактуальные.


## Поддержка внутренних имён библиотеки

После зачистки рядом с исходными файлами могут лежать уже переименованные файлы самой библиотеки.
Stage 1 теперь поддерживает два формата имени:

- исходное имя поставщика Frank RG;
- текущее внутреннее стандартное имя библиотеки.

Внутреннее имя строится по активной схеме из модуля `stratbox.macrobanks.frank_rg.filename_scheme`.
Сейчас используется формат `период_подпись.ext`, а при необходимости в одном месте можно добавить префикс, например `Frank RG_`.

Если в каталоге одновременно лежат:

- исходный файл Frank RG;
- уже переименованный внутренний файл той же семьи и того же периода,

то при отборе `latest` приоритет отдается внутреннему стандартному имени.

Важно:

- функция зачистки не трогает файлы, которые вообще не удалось распознать;
- случайные файлы, битые имена и старые неподдерживаемые схемы остаются как есть;
- удаление затрагивает только корректно распознанные файлы или файлы, признанные слишком старыми по явному правилу семейства.

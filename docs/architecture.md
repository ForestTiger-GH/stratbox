# Архитектура stratbox

`stratbox` — core-репозиторий. Он содержит только библиотечный слой и не содержит surface-код, AppDock connector и GUI.

## Главные слои

- `stratbox.base` — нейтральная инфраструктура;
- `stratbox.common` — общие утилиты;
- `stratbox.macrobanks` — доменные бизнес-модули;
- `stratbox.registries` — встроенные справочники и ресурсные таблицы;
- `stratbox.text` — текстовые нормализаторы.

## Принципы

1. Доменные модули не зависят от desktop-приложения.
2. Репозиторий не содержит AppDock-артефактов.
3. Корпоративные особенности идут через plugin/runtime boundary.
4. Surface-репозитории должны зависеть от `stratbox`, а не копировать его внутрь себя.

## Что не должно попадать в репозиторий

- `src/app`
- `appdock/`
- GUI-зависимости
- runtime/dev-state
- локальные `.venv`
- `.tmp`

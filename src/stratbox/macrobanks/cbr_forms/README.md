# CBR forms (`stratbox.macrobanks.cbr_forms`)

Домен `stratbox.macrobanks.cbr_forms` предназначен для автоматизированной обработки отчетных форм Банка России и формирования сопоставимых Excel-выгрузок по заданному перечню банков и отчетных дат.

Домен решает полный прикладной цикл:

1. формирует ссылки на архивы отчетных форм Банка России;
2. скачивает архивы через общий сетевой слой `stratbox`;
3. распаковывает RAR-архивы во временную рабочую папку;
4. выбирает нужный DBF-файл внутри архива;
5. читает DBF в `pandas.DataFrame`;
6. приводит исходные данные к long-формату;
7. строит итоговую wide-таблицу;
8. сохраняет отдельный Excel-файл по каждой форме.

Домен ориентирован на регулярную работу в Jupyter Notebook, Google Colab, локальной среде и расширенном корпоративном контуре.

---

## Поддерживаемые формы

В текущей версии поддерживаются следующие формы:

| Код запуска | Форма Банка России | Назначение в текущем контуре |
|---|---|---|
| `101` | `0409101` | Балансовые показатели по счетам бухгалтерского учета |
| `102` | `0409102` | Показатели отчета о финансовых результатах |
| `123` | `0409123` | Показатели собственных средств и капитала |
| `135` | `0409135` | Обязательные нормативы банка |
| `805` | `0409805` | Нормативы банковской группы |

Единый реестр форм находится в файле:

```text
src/stratbox/macrobanks/cbr_forms/forms/registry.py
```

Он используется и публичным API, и CLI-скриптом.

---

## Основной публичный API

Главная функция домена:

```python
from stratbox.macrobanks.cbr_forms import run_all_forms_to_xlsx
```

Минимальный пример:

```python
from stratbox.macrobanks.cbr_forms import run_all_forms_to_xlsx

out = run_all_forms_to_xlsx(
    date_from="2024-01-01",
    date_to="2024-03-01",
    out_dir="outputs/cbr_forms",
)

print(out)
```

Функция возвращает словарь:

```python
{
    "101": ".../CBR_0409101_LEGACY.xlsx",
    "102": ".../CBR_0409102_LEGACY.xlsx",
    "123": ".../CBR_0409123_LEGACY.xlsx",
    "135": ".../CBR_0409135_LEGACY.xlsx",
    "805": ".../CBR_0409805_LEGACY.xlsx",
}
```

Если нужно запустить не все формы:

```python
out = run_all_forms_to_xlsx(
    date_from="2024-01-01",
    date_to="2024-03-01",
    forms="101,102,135,805",
    out_dir="outputs/cbr_forms",
)
```

Или так:

```python
out = run_all_forms_to_xlsx(
    date_from="2024-01-01",
    date_to="2024-03-01",
    forms=["135", "805"],
    out_dir="outputs/cbr_forms",
)
```

## Параметры `run_all_forms_to_xlsx`

Основные параметры:

- `date_from` — начальная дата периода;
- `date_to` — конечная дата периода;
- `freq` — частота дат, по умолчанию `"M"`;
- `anchor` — точка периода: `"start"` или `"end"`;
- `banks_mode` — режим списка банков, сейчас поддерживается `"legacy"`;
- `out_dir` — каталог для итоговых Excel-файлов;
- `forms` — перечень форм: `None`, `"all"`, строка через запятую или список строк;
- `timeout`, `retries`, `backoff`, `min_bytes_ok` — сетевые параметры скачивания;
- `show_progress` — показывать индикаторы прогресса.

Пример с датами на начало месяца:

```python
out = run_all_forms_to_xlsx(
    date_from="2024-01-01",
    date_to="2024-06-01",
    freq="M",
    anchor="start",
    forms="all",
    out_dir="outputs/cbr_forms",
)
```

Пример с отключенным прогрессом:

```python
out = run_all_forms_to_xlsx(
    date_from="2024-01-01",
    date_to="2024-06-01",
    forms="135,805",
    out_dir="outputs/cbr_forms",
    show_progress=False,
)
```

## Формирование отчетных дат

Сетка дат строится через общий модуль:

```text
stratbox.common.time.periods
```

Это позволяет использовать единый механизм дат не только в `cbr_forms`, но и в других доменах проекта.

Поддерживаемые частоты:

- `D` — дни;
- `W` — недели;
- `M` — месяцы;
- `Q` — кварталы;
- `Y` — годы.

Для форм Банка России обычно используется месячная частота с `anchor="start"`, так как ссылки на архивы форм строятся по датам вида `YYYYMMDD`.

## Список банков

Сейчас домен поддерживает режим:

```python
banks_mode="legacy"
```

В этом режиме список банков берется из встроенного реестра:

```text
stratbox.registries.cbr_banks
```

Используется колонка `is_legacy`:

- если значение не `False` и не `None`, банк включается в выгрузку;
- числовое значение используется как порядок сортировки;
- регистрационный номер берется из поля `regn`.

Для отображаемого имени банка применяется приоритет:

1. `bank_name_norm` — краткое нормализованное имя;
2. `bank`;
3. `name`;
4. `bank_name`.

В итоговых таблицах по умолчанию используются краткие нормализованные наименования банков. Это сделано, чтобы рабочие Excel-файлы были компактнее и удобнее для регулярного анализа.

## Реестр показателей и формулы

Показатели задаются в CSV-файле:

```text
src/stratbox/macrobanks/cbr_forms/models/formulas.csv
```

Файл содержит:

- код формы (`form`);
- тип обработки (`kind`);
- имя выходного показателя (`name`);
- выражение (`expression`);
- дополнительные параметры (`extra`).

Для форм 101, 102 и 123 используется тип `formula`. Такие показатели рассчитываются по кодам счетов или строк.

Для форм 135 и 805 используется тип `metric`. Такие показатели не рассчитываются как сумма счетов, а берутся напрямую из DBF по коду норматива или метрики.

---

## Формы 101, 102 и 123

Формы 101, 102 и 123 используют расчетную модель:

1. DBF приводится к унифицированной структуре;
2. коды счетов или строк нормализуются;
3. показатели из `formulas.csv` рассчитываются по выражениям;
4. результат собирается в long-таблицу.

Типовая long-таблица содержит поля:

```text
Дата | Банк | Показатель | Значение
```

После этого long-таблица разворачивается в итоговую wide-таблицу.

---

## Формы 135 и 805

Формы 135 и 805 переведены на общий механизм `metric_form.py`.

Это формы метрического типа: значение показателя берется напрямую из DBF по коду норматива.

### Форма 135

Форма `0409135` содержит обязательные нормативы банка.

Ключевые поля DBF для текущей настройки:

- `REGN` — регистрационный номер банка;
- `C1_3` — код норматива;
- `C2_3` — фактическое значение.

Поддерживаемые показатели формы 135:

```text
H1.0, H1.1, H1.2, H1.4, H2, H3, H4, H6, H7, H12, H18, H27, H29
```

### Форма 805

Форма `0409805` содержит нормативы банковской группы.

Ключевые поля DBF для текущей настройки:

- `REGN_GKO` или `REGN` — регистрационный номер головной кредитной организации;
- `NAME_NORM` или `C1_3` — код норматива;
- `FAKT_ZN` или `C2_3` — фактическое значение.

Поддерживаемые показатели формы 805:

```text
H20.0, H20.1, H20.2, H20.4, H22, H26, H28
```

Для формы 805 учтены технические варианты кодов:

- `H20.2` может храниться как `H20_2`;
- `H20.4` может храниться как `H20_4`.

---

## Нормализация кодов нормативов

В отчетности визуально похожие буквы могут быть записаны по-разному:

- латинская `H`;
- кириллическая `Н`;
- верхний или нижний регистр.

Для сопоставления кодов нормативов домен приводит код к единому виду:

- убирает лишние пробелы;
- переводит в верхний регистр;
- заменяет кириллическую `Н` на латинскую `H`;
- применяет технические алиасы, если они заданы для формы.

Это важно для устойчивой обработки форм 135 и 805.

## Выходной формат

Каждая форма сохраняется в отдельный Excel-файл:

```text
CBR_0409101_LEGACY.xlsx
CBR_0409102_LEGACY.xlsx
CBR_0409123_LEGACY.xlsx
CBR_0409135_LEGACY.xlsx
CBR_0409805_LEGACY.xlsx
```

Итоговая таблица строится в wide-формате:

```text
Показатель | Банк | <дата 1> | <дата 2> | ...
```

Сортировка:

- показатели идут в порядке, заданном в `formulas.csv`;
- банки идут в порядке `is_legacy` из реестра банков;
- даты идут в хронологическом порядке.

Экспорт выполняется через:

```text
stratbox.base.ioapi.excel.write_df
```

Для Excel используется стандартный пресет `DEFAULT`. Числовые значения в итоговых таблицах отображаются с разделителем тысячных разрядов, если это задано соответствующим Excel-форматом.

## Сетевой слой и работа внутри контура

Скачивание архивов выполняется через:

```text
stratbox.base.net.download_bytes
```

Это означает, что домен не должен напрямую использовать корпоративные URL-шлюзы. Если установлен и настроен плагин, обработка ссылок может выполняться на уровне общего сетевого слоя.

Формы строят URL в формате:

```text
https://www.cbr.ru/vfs/credit/forms/<form>-<YYYYMMDD>.rar
```

Например:

```text
https://www.cbr.ru/vfs/credit/forms/135-20240101.rar
```

---

## Временные файлы и распаковка RAR

Для обработки архивов домен создает временную рабочую папку через:

```text
stratbox.base.filestore.tmp.make_workdir
```

Архивы RAR распаковываются внешней утилитой:

- `unrar`;
- `7z`;
- `7zz`.

Если ни одна из утилит недоступна, обработка форм, требующих распаковки RAR, завершится ошибкой. В такой ситуации нужно установить `unrar` или `p7zip` в среду выполнения.

DBF-файлы читаются через `dbfread`. Если пакет отсутствует в конкретной среде, его нужно установить отдельно до запуска домена.

---

## Выбор DBF внутри архива

В архивах Банка России может лежать несколько DBF-файлов. Выбор нужного файла выполняется в `common/dbf_picker.py`.

Логика выбора:

1. рекурсивно найти DBF внутри распакованного архива;
2. проверить наличие подходящих полей;
3. применить бонус к скорингу, если имя файла содержит ожидаемую подсказку;
4. выбрать лучший DBF и вернуть layout полей.

Для разных форм задаются свои кандидаты полей. Например:

- форма 135 ожидает `REGN`, `C1_3`, `C2_3`;
- форма 805 ожидает `REGN_GKO/REGN`, `NAME_NORM/C1_3`, `FAKT_ZN/C2_3`.

## Командный запуск

В каталоге домена есть CLI-модуль:

```text
src/stratbox/macrobanks/cbr_forms/run_all_forms.py
```

Он предназначен для запуска обработки из командной строки. Основной сценарий для ноутбуков — всё же импорт `run_all_forms_to_xlsx` и явный вызов функции.

## Структура домена

```text
cbr_forms/
  api.py                 # публичный API для ноутбуков и скриптов
  main.py                # служебный модуль домена
  run_all_forms.py       # CLI-запуск
  run_form101_debug.py   # отладочный запуск формы 101
  common/
    banks.py             # список legacy-банков и отображаемые имена
    dbf.py               # DBF-парсер под выгрузки Банка России
    dbf_picker.py        # выбор подходящего DBF внутри архива
    formulas.py          # загрузка formulas.csv
    metric_form.py       # общий движок форм 135/805
    output.py            # экспорт wide-таблиц в Excel
    runner.py            # скачивание, распаковка и чтение DBF по датам
    wide.py              # построение итоговых wide-таблиц
  forms/
    form101.py
    form102.py
    form123.py
    form135.py
    form805.py
    registry.py          # единый реестр доступных форм
  models/
    formulas.csv         # реестр выходных показателей и выражений
```

## Ограничения текущей версии

1. `banks_mode` сейчас поддерживает только режим `legacy`.
2. Архивы форм должны быть доступны по ссылкам Банка России или через настроенный сетевой слой.
3. Для распаковки RAR нужна внешняя утилита `unrar`, `7z` или `7zz`.
4. Новые формы и новые показатели добавляются через явные реестры, а не через автоматическое угадывание.
5. Для форм 135 и 805 обработка ориентирована на текущую структуру DBF и список нормативов в `formulas.csv`.

---

## Практический смысл домена

`cbr_forms` превращает разрозненные архивы отчетных форм Банка России в единые рабочие Excel-таблицы, пригодные для регулярного анализа банков и банковского сектора.

Ключевые свойства текущей реализации:

- единый запуск по нескольким формам и датам;
- единый список банков;
- единый реестр показателей;
- общий механизм для метрических форм 135/805;
- устойчивость к вариантам написания нормативов через латиницу/кириллицу и регистр;
- воспроизводимый wide-формат итоговых Excel-файлов.

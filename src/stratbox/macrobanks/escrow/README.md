# Escrow (`stratbox.macrobanks.escrow`)

Домен `stratbox.macrobanks.escrow` предназначен для обработки ежемесячных Excel-публикаций по финансированию долевого строительства и счетам эскроу. Он формирует единую историческую Excel-витрину по всем доступным датам, сохраняя устойчивую структуру листов и сопоставимость показателей.

Домен решает две основные задачи:

1. автоматически получить и разобрать исходные месячные публикации;
2. собрать итоговую книгу с фиксированными показателями, стабильными именами листов и воспроизводимым порядком строк.

---

## Место в архитектуре

`escrow` — специализированный домен внутри `stratbox.macrobanks`. Он не реализует собственный файловый транспорт и не дублирует инфраструктурные сервисы проекта.

Используются общие слои `stratbox`:

- `stratbox.base.net` — получение страницы источника и скачивание Excel-файлов;
- `FileStore` — единый доступ к локальным и расширенным файловым хранилищам;
- `ioapi.bytes` и `ioapi.zip` — сохранение исходников и итогового архива;
- `openpyxl` — сборка итоговой Excel-книги.

За счет этого один и тот же доменный код может работать:

- в локальной среде;
- в Google Colab;
- в Jupyter Notebook;
- в расширенной корпоративной среде с плагином;
- с локальными путями или сетевым диском через активный `FileStore`.

---

## Функциональный конвейер

Полный сценарий обработки включает следующие этапы:

1. загрузка страницы источника со списком ежемесячных Excel-файлов;
2. извлечение ссылок на исходные публикации;
3. скачивание файлов через общий сетевой слой `stratbox`;
4. опциональное сохранение исходных файлов в кэш;
5. разбор каждого Excel-файла в нормализованный long-поток;
6. объединение исторических данных по всем датам;
7. построение сводных таблиц по каждому показателю;
8. сборка итоговой Excel-книги;
9. сохранение результата как `.xlsx` или как `.zip` с `.xlsx` внутри.

---

## Принципы парсинга

### Иерархический разбор строк

Исходная таблица не является простым плоским списком регионов. В ней встречаются строки разных типов:

- федеральный округ;
- субъект Российской Федерации;
- итог по Российской Федерации.

Парсер явно определяет тип строки, отображаемое имя и исходное положение строки. Порядок сохраняется через `display_order`, чтобы итоговая книга воспроизводила понятную последовательность из источника.

### Реестр показателей

Показатели не определяются случайными эвристиками. В домене используется явный реестр известных показателей:

- каноническое имя;
- код листа;
- правила распознавания заголовка;
- порядок в выходной книге.

Такой подход позволяет:

- сохранять стабильные имена листов;
- контролировать изменения структуры источника;
- спокойно обрабатывать исторические сноски и варианты заголовков;
- расширять домен при появлении новых показателей.

### Единый выходной контракт

Итоговая книга строится по полному упорядоченному набору известных показателей. Если в старых файлах часть показателей отсутствует, это не ломает обработку: по таким датам значения остаются пустыми.

Это важно для исторических рядов: структура итогового файла должна быть стабильной, даже если структура исходников менялась во времени.

---

## Поддерживаемые показатели

В текущей версии поддерживаются следующие листы итоговой книги:

| Код листа | Показатель |
|---|---|
| `КДКД` | Кол-во действующих кредитных договоров |
| `СДКДМР` | Сумма действующих кредитных договоров, млн руб. |
| `ЗМР` | Задолженность, млн руб. |
| `КСЭ` | Кол-во счетов эскроу |
| `КСЭИО` | Кол-во счетов эскроу, имеющих остатки |
| `ОСНСЭМР` | Остатки средств на счетах эскроу, млн руб. |
| `ССПКДПФО` | Средневзвешенная ставка по кредитным договорам по федеральному округу, % |
| `КРСЭ` | Кол-во раскрытых счетов эскроу |
| `ССПСРСЭМР` | Сумма средств, перечисленных с раскрытых счетов эскроу, млн руб. |

Коды листов являются частью выходного контракта домена. Их не следует менять без отдельной миграции, так как на них могут опираться рабочие ссылки в Excel-файлах.

---

## Устойчивость к историческим форматам

Публикации источника встречаются в нескольких исторических вариантах. Домен рассчитан на обработку старых, промежуточных и новых форматов.

### Варианты заголовков

Распознавание заголовков выполняется после нормализации текста. Нормализация учитывает:

- сноски и цифровые хвосты;
- примечания в скобках;
- типографские кавычки;
- разные пробелы и знаки препинания;
- исторические варианты формулировок, включая различие между `Средняя ставка...` и `Средневзвешенная ставка...`.

### Варианты итоговой строки

Поддерживаются разные подписи итоговой строки, в том числе:

- `Итого`;
- `Итого по РФ`;
- `Итого по Российской Федерации`.

После итоговой строки разбор данных прекращается. Служебные сноски и текстовые хвосты ниже итоговой строки не попадают в поток данных.

### Отсутствующие показатели

Для конкретной исторической даты часть известных показателей может отсутствовать. Это считается нормальным состоянием источника и не останавливает обработку.

При этом действительно новые неизвестные показатели должны добавляться в реестр явно. Домен не пытается автоматически включать новые столбцы в итоговую книгу.

## Публичный API

Основной публичный вход:

```python
from stratbox.macrobanks.escrow import run_escrow_to_xlsx
```

Функция:

```python
run_escrow_to_xlsx(...)
```

возвращает `EscrowRunResult`.

---

## Основные параметры

Наиболее важные параметры `run_escrow_to_xlsx`:

- `out_path` — путь к итоговому `.xlsx` или `.zip` через активный `FileStore`;
- `archive` — если `True`, результат сохраняется как ZIP-архив;
- `archive_member_name` — имя Excel-файла внутри ZIP;
- `source_cache_dir` — папка кэша исходных Excel-файлов;
- `refresh` — если `True`, исходники скачиваются заново даже при наличии кэша;
- `index_url` — адрес страницы источника со списком файлов;
- `regions_mode` — режим порядка строк итоговой витрины;
- `custom_regions` — пользовательский порядок строк при `regions_mode="custom"`;
- `timeout`, `retries`, `backoff`, `min_bytes_ok`, `headers` — сетевые параметры;
- `plugin_only` — режим обработки URL через инфраструктурный слой;
- `show_progress` — отображение прогресса;
- `filestore` — явный экземпляр `FileStore`, если нужно передать его вручную.

### Режимы `regions_mode`

Поддерживаются:

- `latest` — порядок строк берётся из последнего доступного файла;
- `custom` — используется пользовательский список строк;
- `registry` — зарезервирован под будущий реестр регионов.

---

## Результат выполнения

`EscrowRunResult` содержит служебную информацию:

- `output_path` — путь к итоговому файлу;
- `archive` — был ли создан ZIP;
- `source_files` — список исходных файлов;
- `dates` — даты, вошедшие в итоговую витрину;
- `indicators` — список показателей;
- `regions` — порядок строк;
- `rows_long` — размер объединённого long-потока.

---

## Пример сохранения XLSX

```python
from stratbox.macrobanks.escrow import run_escrow_to_xlsx

result = run_escrow_to_xlsx(
    out_path="outputs/Escrow Accounts.xlsx",
    source_cache_dir="cache/escrow",
    archive=False,
)

print(result.output_path)
print(result.dates)
```

## Пример сохранения ZIP

```python
from stratbox.macrobanks.escrow import run_escrow_to_xlsx

result = run_escrow_to_xlsx(
    out_path="outputs/Escrow Accounts.zip",
    archive=True,
    source_cache_dir="cache/escrow",
)

print(result.output_path)
```

## Кэш исходных файлов

Если задан `source_cache_dir`, каждый скачанный Excel-файл сохраняется через активный `FileStore`.

Это полезно для:

- повторных запусков;
- отладки парсера;
- хранения копии исходников;
- работы с сетевым диском без отдельной Samba-логики.

Если `refresh=False` и файл уже есть в кэше, домен читает его из кэша без повторного скачивания.

## Структура домена

```text
escrow/
  api.py       # публичный API и общий конвейер
  sources.py   # получение ссылок на исходные файлы
  parser.py    # разбор Excel-файлов в long-формат
  rows.py      # распознавание строк: округ, регион, итог
  columns.py   # реестр и распознавание показателей
  pivots.py    # построение сводных таблиц
  workbook.py  # сборка итоговой Excel-книги
  output.py    # сохранение XLSX/ZIP через FileStore
  regions.py   # режимы порядка регионов
  models.py    # модели данных домена
```

---

## Ограничения текущей версии

1. Домен ориентирован на стандартные публикации источника по счетам эскроу.
2. Новые неизвестные показатели добавляются в реестр вручную.
3. `regions_mode="registry"` зарезервирован, но не реализован как полноценный внешний реестр.
4. Форматирование итоговой книги отражает текущую рабочую витрину и может быть позднее унифицировано с общими Excel-пресетами `stratbox`.

## Практический результат

Текущая реализация дает устойчивый рабочий конвейер:

- скачивание и кэширование исходников;
- обработка нескольких исторических поколений файлов;
- нормализованный long-поток;
- стабильные итоговые листы;
- сохранение как Excel или ZIP;
- переносимость между локальной и расширенной корпоративной средой.

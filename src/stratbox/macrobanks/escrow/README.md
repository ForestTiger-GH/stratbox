# Escrow domain

Домен `stratbox.macrobanks.escrow` предназначен для обработки ежемесячных Excel-файлов Банка России по финансированию долевого строительства (счета эскроу) и формирования единой выходной Excel-витрины по всем доступным датам.

Домен решает две прикладные задачи:

1. автоматизированное получение и разбор ежемесячных публикаций Банка России;
2. формирование устойчивого выходного файла с фиксированной структурой листов и сохранением исторической сопоставимости данных.

## Назначение и место в проекте

`escrow` является специализированным доменом внутри `stratbox.macrobanks`. Он не дублирует общую инфраструктуру проекта, а использует базовые сервисы `stratbox`:

- сетевой слой `stratbox.base.net` для получения исходных файлов;
- `FileStore` и `ioapi.bytes/zip` для чтения и записи файлов одинаковым образом во внешнем и внутреннем контуре;
- `openpyxl`-сборку книги для финальной выходной витрины.

За счёт этого домен работает единообразно:

- локально вне банковского контура;
- в среде с подключённым `stratbox-plugin`;
- с локальным хранилищем или сетевым диском через активный `FileStore`.

Домен не содержит отдельной логики `INBANK` и не работает «в обход» инфраструктуры `stratbox`.

## Функциональный сценарий

Полный конвейер обработки включает следующие этапы:

1. чтение страницы Банка России со списком ежемесячных Excel-файлов;
2. извлечение ссылок на исходные публикации;
3. скачивание файлов через общий сетевой слой `stratbox`;
4. при необходимости — сохранение исходных файлов в каталог кэша через `FileStore`;
5. разбор каждого Excel-файла в семантически корректный поток данных;
6. объединение исторического ряда по всем датам;
7. построение сводных таблиц по каждому показателю;
8. формирование итоговой Excel-книги;
9. сохранение результата как `.xlsx` либо `.zip` с `.xlsx` внутри.

## Архитектурные принципы

### 1. Иерархический разбор строк

Исходная таблица Банка России не является плоским перечнем регионов. Она содержит несколько различных типов строк:

- федеральный округ;
- субъект Российской Федерации;
- итог по Российской Федерации.

Парсер обрабатывает эту структуру явно. Для каждой строки определяется её тип, отображаемое имя и положение в исходной таблице. Порядок строк сохраняется через `display_order`, что позволяет воспроизводить в итоговой книге ту же последовательность, что и в источнике.

### 2. Отдельный реестр показателей

Заголовки показателей не интерпретируются «на лету» по первым буквам или по случайному набору слов. Для домена используется фиксированный реестр известных показателей с каноническими именами, правилами распознавания и кодами листов.

Это обеспечивает:

- стабильность имён листов в итоговой книге;
- устойчивость к сноскам, постфиксам и примечаниям в заголовках;
- возможность явно расширять реестр при изменении структуры формы.

### 3. Единый выходной контракт

Итоговая книга строится по полному упорядоченному реестру выходных листов. Даже если в старых исторических файлах часть показателей отсутствует, структура выходной книги остаётся единой, а отсутствующие по соответствующим датам значения остаются пустыми.

### 4. Явное расширение при изменении источника

Если Банк России добавляет новый показатель, его необходимо явно включить в реестр домена. Такой подход принят сознательно: редкие изменения структуры источника должны обрабатываться не эвристически, а контролируемо и прозрачно.

## Поддерживаемые исторические варианты структуры

На практике публикации Банка России по счетам эскроу встречаются минимум в нескольких исторических вариантах.

Домен поддерживает:

- старый формат, в котором отсутствуют отдельные позднее добавленные показатели;
- промежуточный формат;
- новый формат, в котором присутствуют дополнительные показатели и примечания в заголовках.

Парсер допускает исторически неполный набор известных показателей в конкретном месяце. Это считается нормальным состоянием формы и не приводит к остановке обработки.

Одновременно сохраняется жёсткий контроль над действительно новыми неучтёнными столбцами: такие изменения должны добавляться в реестр осознанно.

## Поддерживаемые показатели и коды листов

В текущей версии домена поддерживаются следующие показатели:

| Код листа | Показатель |
|---|---|
| `КДКД` | Кол-во действующих кредитных договоров |
| `СДКДМР` | Сумма действующих кредитных договоров, млн руб. |
| `ЗМР` | Задолженность, млн руб. |
| `КСЭ` | Кол-во счетов эскроу |
| `КСЭИО` | Кол-во счетов эскроу, имеющих остатки |
| `ОСНСЭМР` | Остатки средств на счетах эскроу, млн руб. |
| `ССПКДПФО` | Средневзвешенная ставка по кредитным договорам по федеральному округу, % |
| `КРСЭ` | Кол-во «раскрытых» счетов эскроу |
| `ССПСРСЭМР` | Сумма средств, перечисленных с «раскрытых» счетов эскроу, млн руб. |

Коды листов фиксированы и используются как часть рабочего контракта выходного файла.

## Устойчивость к вариативности заголовков

Распознавание заголовков выполняется после нормализации текста. Нормализация специально игнорирует:

- сноски и хвостовые цифровые пометки;
- примечания в круглых скобках;
- типографские кавычки;
- технические различия пробелов и знаков препинания.

Это позволяет одинаково обрабатывать старые и новые публикации при сохранении строгого контроля по известным показателям.

## Устойчивость к вариативности итоговой строки

Исторические файлы используют разные варианты подписи итоговой строки. Домен корректно распознаёт, в частности, следующие варианты:

- `Итого`;
- `Итого по РФ`;
- `Итого по Российской Федерации`.

После нахождения итоговой строки разбор данных прекращается. Сноски и служебный хвост ниже итоговой строки в обработку не включаются.

## Публичный API

Основной публичный вход домена:

```python
from stratbox.macrobanks.escrow import run_escrow_to_xlsx
```

Главная функция:

```python
run_escrow_to_xlsx(...)
```

Она запускает полный конвейер обработки и возвращает объект `EscrowRunResult`.

## Параметры `run_escrow_to_xlsx`

Наиболее важные параметры:

- `out_path` — путь к итоговому файлу через активный `FileStore`;
- `archive` — если `True`, результат сохраняется как `.zip`, внутри которого лежит `.xlsx`;
- `archive_member_name` — имя `.xlsx` внутри архива;
- `source_cache_dir` — каталог для сохранения исходных месячных Excel-файлов;
- `refresh` — если `True`, кэш исходников пересоздаётся заново;
- `index_url` — адрес страницы Банка России со списком файлов;
- `regions_mode` — режим формирования списка строк витрины;
- `custom_regions` — пользовательский порядок строк при `regions_mode='custom'`;
- `timeout`, `retries`, `backoff`, `min_bytes_ok`, `headers` — сетевые параметры скачивания;
- `plugin_only` — режим сетевого слоя при наличии инфраструктурного плагина;
- `show_progress` — показывать ли индикаторы прогресса;
- `filestore` — явный экземпляр `FileStore`, если требуется передать его вручную.

### Режимы `regions_mode`

Поддерживаются:

- `latest` — порядок строк берётся по последнему доступному файлу;
- `custom` — используется пользовательский список строк;
- `registry` — зарезервирован на будущее, под подключение отдельного реестра регионов.

## Результат выполнения

`run_escrow_to_xlsx()` возвращает `EscrowRunResult`, содержащий краткую служебную информацию о выполнении конвейера:

- `output_path` — путь к итоговому файлу;
- `archive` — был ли создан архив;
- `source_files` — список обработанных исходных файлов;
- `dates` — даты, вошедшие в итоговую витрину;
- `indicators` — список выходных показателей;
- `regions` — порядок строк витрины;
- `rows_long` — размер объединённого длинного потока данных.

## Минимальный пример

```python
from stratbox.macrobanks.escrow import run_escrow_to_xlsx

result = run_escrow_to_xlsx(
    out_path="outputs/Escrow Accounts.xlsx",
    source_cache_dir="cache/escrow",
    archive=False,
)

print(result.output_path)
print(result.dates)
```

## Пример сохранения в ZIP

```python
from stratbox.macrobanks.escrow import run_escrow_to_xlsx

result = run_escrow_to_xlsx(
    out_path="outputs/Escrow Accounts.zip",
    archive=True,
    source_cache_dir="cache/escrow",
)

print(result.output_path)
```

## Пример работы с локальным режимом вне банковского контура

При необходимости можно явно отключить использование инфраструктурного плагина и задать локальный корень `stratbox` в окружении.

```python
import os

os.environ["STRATBOX_USE_PLUGIN"] = "0"
os.environ["STRATBOX_LOCAL_ROOT"] = "/content/stratbox_work"
```

После этого домен будет использовать локальный `FileStore`, а все пути будут интерпретироваться относительно локального окружения.

## Сохранение исходных файлов

Если указан `source_cache_dir`, каждый скачанный Excel-файл сохраняется в кэш через активный `FileStore`. Это позволяет:

- повторно использовать уже скачанные исходники;
- ускорять отладку и повторный прогон;
- хранить копию месячных файлов на сетевом диске или локально без отдельной логики для разных сред.

Если `refresh=False`, при наличии файла в кэше он будет прочитан из `FileStore` без повторного скачивания.

## Ограничения текущей версии

1. Домен ориентирован на стандартную форму публикации Банка России по счетам эскроу.
2. Новые неизвестные показатели не добавляются автоматически и должны включаться в реестр явно.
3. `regions_mode='registry'` пока не реализован и зарезервирован на будущее.
4. Форматирование итоговой книги пока отражает текущую рабочую витрину и может быть приведено к более общему стандарту `stratbox` позднее.

## Практический смысл текущей реализации

Текущая реализация обеспечивает следующее:

- единый парсер для нескольких исторических поколений файлов;
- стабильную выходную книгу с фиксированными кодами листов;
- одинаковую работу во внешнем и внутреннем контуре;
- контролируемое расширение домена при изменениях структуры источника.

Именно это делает домен пригодным не только для разовой выгрузки, но и для регулярного производственного использования внутри проекта `macrobanks`.

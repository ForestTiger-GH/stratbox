# FRG (`stratbox.macrobanks.frg`)

Домен `stratbox.macrobanks.frg` предназначен для первого файлового этапа работы с регулярными файлами FRG. На текущем этапе домен не читает содержимое Excel-файлов и не выполняет предметный парсинг таблиц. Его задача — привести файловый контур в устойчивое и воспроизводимое состояние перед дальнейшей обработкой.

Домен решает следующие задачи:

1. сканирует каталог с файлами;
2. распознает семейство файла по имени;
3. извлекает и нормализует период;
4. строит полный каталог файлов;
5. выбирает наиболее актуальный файл по каждому семейству;
6. готовит план зачистки каталога;
7. при необходимости копирует или переименовывает актуальные файлы;
8. удаляет только допустимые устаревшие распознанные файлы;
9. собирает ZIP-архив актуального набора.

---

## Главный принцип

`frg` работает с файловым контуром, а не с содержимым таблиц.

Домен не выполняет:

- чтение листов Excel;
- изменение содержимого книг;
- пересохранение Excel-файлов;
- расчет показателей;
- предметный парсинг таблиц.

Все решения принимаются на основе:

- имени файла;
- расширения;
- пути;
- базовых метаданных файла;
- правил распознавания, заданных в реестре.

Это делает домен безопасным для использования в общей папке с исходными поставками.

---

## Работа с каталогом

По умолчанию домен сканирует только верхний уровень каталога.

Это сделано специально, чтобы не затрагивать вложенные папки, например:

- `Архив`;
- служебные подпапки;
- ручные рабочие каталоги пользователя.

Файлы во вложенных каталогах не сканируются, не переименовываются, не удаляются и не попадают в итоговый архив, если явно не включать рекурсивный режим в отдельных функциях сканирования.

---

## Безопасность нераспознанных файлов

Если файл не соответствует ни одной поддерживаемой схеме имени, он не считается ошибкой выполнения.

Нераспознанные файлы:

- не переименовываются;
- не копируются как актуальные;
- не удаляются;
- не попадают в архив актуального набора.

Это принципиальное правило домена. Оно позволяет запускать обработку в рабочих папках, где могут лежать случайные или вспомогательные файлы.

---

## Поддерживаемые семейства

Реестр семейств находится в файле:

```text
src/stratbox/macrobanks/frg/registry.py
```

На текущем этапе поддерживаются следующие семейства:

| Код семейства | Семейство |
|---|---|
| `express_issuance` | Экспресс выдачи |
| `express_portfolios` | Экспресс портфели |
| `express_passives` | Экспресс — Пассивы |
| `express_issuance_weekly` | Экспресс выдачи — Weekly |
| `express_issuance_cards` | Экспресс выдачи — Карты |
| `regional_issuance` | Региональные выдачи |
| `regional_portfolios` | Региональные портфели |
| `regional_issuance_mortgage` | Региональные выдачи — Ипотека |
| `regional_portfolios_mortgage` | Региональные портфели — Ипотека |
| `regional_issuance_cards` | Региональные выдачи — Карты |
| `rbm_volumes_q` | Выдачи — Q |
| `rbm_portfolios_q` | Портфели — Q |
| `volumes_cards_q` | Выдачи — Карты — Q |
| `mortgage_refinancing` | Рефинансирование ипотеки |

Порядок правил важен: более специфичные семейства должны идти раньше более общих. Например, ипотечные региональные файлы должны распознаваться до общих региональных выдач.

---

## Поддерживаемые расширения

По умолчанию в каталог включаются файлы с расширениями:

```text
.xlsx, .xlsb, .xlsm, .xls
```

Файлы с другими расширениями могут попасть в каталог как найденные объекты, но будут помечены как неподдерживаемые.

---

## Схемы имен файлов

Домен поддерживает две схемы именования:

1. исходные имена поставщика;
2. внутренние имена, которые уже сформировала библиотека после предыдущей зачистки.

Это позволяет безопасно запускать домен повторно в одной и той же папке, где одновременно могут лежать:

- новые файлы поставщика;
- ранее созданные внутренние файлы;
- нераспознанные или ручные файлы пользователя.

Если по одному семейству и периоду есть и исходное имя, и внутреннее стандартное имя, приоритет отдается внутреннему имени.

---

## Нормализация периодов

Домен извлекает период из имени файла и приводит его к стандартному виду.

### Календарные даты

Поддерживаются даты с разными разделителями:

- `.`;
- `-`;
- `_`;
- `/`.

Итоговый стандарт:

```text
YYYY-MM-DD
```

### Недельные файлы

Для weekly-файлов поддерживается формат вида:

```text
YYYY-MM week N
```

Примеры входных вариантов:

- `Май 2026 1 неделя`;
- `2026 Февраль 3 неделя`;
- `Выдачи - Май 2026 2 недели`.

Итоговая нормализация:

```text
2026-05 week 1
2026-02 week 3
```

---

## Правило актуальности weekly-файлов

Для weekly-файлов действует специальная логика: внутри одного месяца актуальной считается самая поздняя неделя.

Если в каталоге есть:

- `2026-05 week 1`;
- `2026-05 week 2`;
- `2026-05 week 3`,

то `week 1` и `week 2` помечаются как устаревшие по причине:

```text
superseded_by_newer_week_in_same_month
```

В режиме зачистки такие файлы могут быть удалены, если включено `delete_others=True`.

---

## Правило по региональным файлам

Для региональных файлов зафиксирована новая схема начиная с:

```text
2026-04-01
```

Более ранние региональные файлы:

- остаются видимыми в полном каталоге;
- не попадают в `latest`;
- могут быть удалены в режиме зачистки как устаревшие по правилу семейства.

Причина:

```text
period_older_than_family_min_date
```

---

## Внутренняя схема имен

После зачистки домен может создать внутренние стандартные имена.

Активная схема:

```text
период_подпись.ext
```

Примеры:

```text
2026-05-01_Экспресс выдачи.xlsx
2026-05 week 2_Экспресс выдачи — Weekly.xlsx
2026-05-15_Экспресс — Пассивы.xlsb
```

Схема задается централизованно в файле:

```text
src/stratbox/macrobanks/frg/filename_scheme.py
```

Если в будущем потребуется изменить стандарт имен, это должно делаться как отдельная управляемая миграция.

## Публичный API

### Сканирование и выбор актуальных файлов

```python
from stratbox.macrobanks.frg import run_frg_stage1

result = run_frg_stage1("data/frg")

catalog_df = result["catalog"]
latest_df = result["latest"]
dispatch_df = result["dispatch"]
```

Возвращаемые таблицы:

- `catalog` — полный каталог найденных файлов;
- `latest` — по одному наиболее актуальному файлу на семейство;
- `dispatch` — журнал диспетчеризации в заглушки будущих парсеров.

### Построение каталога отдельно

```python
from stratbox.macrobanks.frg import build_frg_catalog

catalog_df = build_frg_catalog("data/frg")
```

### Отбор latest отдельно

```python
from stratbox.macrobanks.frg import select_latest_frg_files

latest_df = select_latest_frg_files(catalog_df)
```

## Зачистка каталога

Главная функция зачистки:

```python
from stratbox.macrobanks.frg import run_frg_cleanup
```

Безопасный просмотр плана без выполнения:

```python
result = run_frg_cleanup(
    "data/frg",
    delete_others=False,
    archive_latest=False,
    execute=False,
)

plan_df = result["plan"]
```

Функция возвращает:

- `catalog` — полный каталог;
- `latest` — актуальные файлы;
- `plan` — план файловых действий;
- `execution` — журнал выполнения, если `execute=True`.

### Режим `delete_others=False`

В этом режиме домен:

- определяет актуальные файлы;
- строит стандартные внутренние имена;
- при `execute=True` копирует актуальные файлы рядом с исходными;
- не удаляет исходные файлы.

Такой режим удобен для аккуратного первого запуска.

### Режим `delete_others=True`

В этом режиме домен:

- переименовывает актуальные файлы в стандартный внутренний вид;
- удаляет распознанные, но неактуальные файлы;
- удаляет вытесненные weekly-файлы;
- удаляет региональные файлы, признанные слишком старыми по правилу семейства.

При этом домен не удаляет:

- нераспознанные файлы;
- случайные файлы;
- вложенные каталоги;
- содержимое вложенных каталогов.

---

## Архивирование актуального набора

Для создания ZIP-архива актуальных файлов используется параметр:

```python
archive_latest=True
```

Пример:

```python
result = run_frg_cleanup(
    "data/frg",
    delete_others=True,
    archive_latest=True,
    execute=True,
)
```

Архив содержит только актуальные `latest`-файлы.

Имя архива:

```text
FRG_Actuals_YYYY-MM-DD.zip
```

Например:

```text
FRG_Actuals_2026-05-27.zip
```

Архив сохраняется в тот же каталог верхнего уровня. Он не участвует в распознавании семейств и не влияет на следующий выбор `latest`.

## Структура домена

```text
frg/
  api.py              # публичный API
  catalog.py          # построение полного каталога
  cleanup.py          # план и выполнение зачистки, архивирование
  dispatch.py         # диспетчеризация latest-файлов в будущие парсеры
  filename_scheme.py  # активная схема внутренних имен
  models.py           # модели правил и записей каталога
  naming.py           # распознавание исходных и внутренних имен
  registry.py         # реестр семейств FRG
  selection.py        # выбор актуальных файлов
  parsers/            # заглушки будущих парсеров
  run_scan_debug.py   # служебный отладочный запуск
```

## Практический сценарий работы

### 1. Посмотреть каталог

```python
from stratbox.macrobanks.frg import run_frg_stage1

result = run_frg_stage1("data/frg")
result["catalog"].head()
```

### 2. Проверить latest

```python
result["latest"]
```

### 3. Построить план зачистки

```python
from stratbox.macrobanks.frg import run_frg_cleanup

result = run_frg_cleanup(
    "data/frg",
    delete_others=True,
    archive_latest=True,
    execute=False,
)

result["plan"]
```

### 4. Выполнить план

```python
result = run_frg_cleanup(
    "data/frg",
    delete_others=True,
    archive_latest=True,
    execute=True,
    replace_existing=True,
)

result["execution"]
```

## Ограничения текущей версии

1. Домен не парсит содержимое Excel-файлов.
2. Распознавание зависит от правил в `registry.py` и качества имен файлов.
3. По умолчанию обрабатывается только верхний уровень каталога.
4. Поддерживается одна активная внутренняя схема имен.
5. Нераспознанные файлы сознательно не трогаются.

---

## Практический результат

Текущая реализация обеспечивает устойчивый файловый слой для дальнейшей обработки FRG:

- каталог файлов становится проверяемым;
- актуальные версии выбираются автоматически;
- устаревшие файлы можно безопасно убрать;
- итоговый набор можно сохранить как ZIP;
- повторный запуск в том же каталоге поддерживается за счет распознавания внутренних имен.

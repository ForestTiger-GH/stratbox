# `stratbox.base`

`stratbox.base` — нейтральный инфраструктурный слой библиотеки. Он не содержит доменной логики по банковским формам, FRG или эскроу, а предоставляет общие механизмы, на которых эти домены работают.

Главная задача слоя — скрыть различия между локальной средой, Colab, Jupyter и расширенной корпоративной средой за едиными интерфейсами.

## Состав каталога

```text
base/
  filestore/     # единый транспорт файлов и каталогов
  ioapi/         # чтение и запись форматов поверх FileStore
  net/           # скачивание и обработка URL
  secrets/       # интерфейс получения секретов
  styles/excel/  # Excel-пресеты и применение оформления
  utils/         # небольшие технические утилиты
  runtime.py     # выбор активных provider'ов: plugin или local
```

## FileStore

`FileStore` — основной файловый контракт `stratbox`. Он описывает операции:

- открыть файл на чтение или запись;
- проверить существование файла или каталога;
- получить метаданные;
- создать каталог;
- удалить файл или каталог;
- переименовать или переместить объект;
- прочитать или записать bytes;
- обойти каталог через `walk`;
- найти файлы по маске через `glob`.

В стандартной среде используется `LocalFileStore`. В расширенной корпоративной среде активный `FileStore` может быть предоставлен плагин.

Доменный код должен опираться именно на этот интерфейс, а не на конкретную файловую систему.

## IO API

`ioapi` — набор форматных обёрток поверх `FileStore`.

Рекомендуемый импорт:

```python
from stratbox.base import ioapi as ia
```

Примеры:

```python
# Excel
ia.excel.write_df("outputs/table.xlsx", df)
df = ia.excel.read_df("outputs/table.xlsx")

# bytes
content = ia.bytes.read_bytes("data/file.bin")
ia.bytes.write_bytes("data/copy.bin", content)

# ZIP из памяти
ia.zip.write_zip_from_memory(
    "outputs/archive.zip",
    {"file_1.txt": b"hello", "file_2.txt": b"world"},
)
```

Поддерживаются модули для Excel, CSV, DBF, XML, TXT, ZIP/RAR, PDF, DOCX, PPTX и изображений в пределах реализованного функционала.

## Сетевой слой

`base.net` отвечает за сетевое получение файлов и обработку URL.

Главные функции:

```python
from stratbox.base.net import download_bytes, url
```

`download_bytes()` возвращает структурированный результат скачивания: статус, bytes-содержимое, итоговый URL, ошибку и HTTP-заголовки.

URL перед скачиванием проходит через `stratbox.base.net.url.url()`. Это позволяет во внешней среде использовать обычные публичные ссылки, а в корпоративной среде — подменять их через плагин и внутренний шлюз без изменения доменного кода.

## Runtime и провайдеры

`runtime.py` — единственная точка выбора активных провайдеров:

- если доступен и включен плагин, используются его провайдеры;
- иначе включается локальный режим.

Основные функции:

```python
from stratbox.base.runtime import get_filestore, get_secrets, get_providers

store = get_filestore()
secrets = get_secrets()
```

Отключить plugin-режим можно переменной окружения:

```python
import os
os.environ["STRATBOX_USE_PLUGIN"] = "0"
```

Задать локальный корень для относительных путей можно так:

```python
import os
os.environ["STRATBOX_LOCAL_ROOT"] = "/content/stratbox_work"
```

## Секреты

Слой `secrets` задаёт общий интерфейс `SecretProvider`. В локальном режиме используются переменные окружения и интерактивный prompt. В расширенной среде провайдер секретов может быть подменён плагином.

Домены не должны самостоятельно реализовывать получение паролей и токенов, если это можно сделать через общий provider.

## Excel-стили

`base.styles.excel` содержит систему Excel-пресетов.

Смысл слоя:

- хранить встроенные пресеты оформления;
- принимать пресеты из plugin-слоя;
- применять стиль к уже записанному workbook через `openpyxl`;
- централизованно управлять внешним видом рабочих таблиц.

Получить список доступных пресетов:

```python
from stratbox.base.styles.excel.main import list_available_presets

print(list_available_presets())
```

## Временные каталоги

`filestore.tmp` содержит утилиты для временных рабочих папок:

```python
from stratbox.base.filestore.tmp import make_workdir, workdir

with workdir("cbr_forms_") as wd:
    ...
```

Это полезно для промежуточных операций: скачивание архивов, распаковка, подготовка файлов перед загрузкой через `FileStore`.

## Правила использования слоя

1. Доменный код должен использовать `FileStore/ioapi` для устойчивой работы в разных средах.
2. Скачивание внешних файлов должно идти через `base.net.download_bytes`.
3. Прямой импорт плагин из доменов запрещён по смыслу архитектуры.
4. Корпоративные особенности должны подключаться через provider'ы runtime.
5. Форматные операции должны жить в `ioapi`, а не размазываться по доменным модулям.

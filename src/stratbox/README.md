# `src/stratbox`

Каталог `src/stratbox` содержит основной пакет библиотеки `stratbox`.

Пакет разделён на несколько слоёв:

- `base` — нейтральная инфраструктура: файловый транспорт, IO API, сетевой слой, runtime-провайдеры, секреты и Excel-стили;
- `common` — общие утилиты без привязки к конкретному домену;
- `macrobanks` — прикладные домены для макроэкономических задач;
- `registries` — встроенные справочники и ресурсные таблицы;
- `text` — вспомогательные текстовые нормализаторы.

## Основной принцип

`stratbox` строится как библиотека, где доменная логика отделена от физической среды исполнения.

Код доменов не должен напрямую зависеть от того, где выполняется ноутбук или скрипт:

- локально;
- в Google Colab;
- в корпоративном Jupyter;
- с подключённым сетевым диском;
- с установленным плагином.

Для этого инфраструктурные операции вынесены в `base`:

- чтение и запись файлов идут через `FileStore` и `ioapi`;
- скачивание URL идёт через `base.net`;
- выбор локальных провайдеров или провайдеров плагина выполняется в `base.runtime`.

## Рекомендуемые импорты

Инфраструктурный IO-слой:

```python
from stratbox.base import ioapi as ia
```

Доменные функции:

```python
from stratbox.macrobanks.cbr_forms import run_all_forms_to_xlsx
from stratbox.macrobanks.escrow import run_escrow_to_xlsx
from stratbox.macrobanks.frg import run_frg_stage1, run_frg_cleanup
from stratbox.macrobanks.cbr_archiver import CbrSourceCollectRequest, collect_cbr_sources
```

## Что не должно попадать в основной пакет

В основной пакет `stratbox` не следует добавлять:

- пароли, логины, токены и иные секреты;
- прямые корпоративные импорты;
- прямую зависимость доменного кода от Samba или конкретного сетевого диска;
- одноразовые ноутбучные фрагменты, которые не оформлены как повторно используемые функции.

Корпоративные особенности должны подключаться через отдельный плагин и стандартные точки расширения.
